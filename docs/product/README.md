# Product Brief Contract

Use `BRIEF.md` for product intent that drives planning and decomposition.

For MyClaw, keep this aligned with:

- `README.md` user-facing goals
- `docs/SPEC.md` behavior contract
- `docs/REQUIREMENTS.md` product and architecture intent

`BRIEF.md` should include:

- problem statement
- scope and non-goals
- user outcomes
- acceptance criteria for the active feature run
